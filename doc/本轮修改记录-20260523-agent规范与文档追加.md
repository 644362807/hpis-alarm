# 本轮修改记录：agent 规范与文档追加

本轮需求要求为 `hpis-alarm` 增加 Agent 协作规范，明确后续每次代码修改都必须补充详细中文注释，并且每轮修改必须生成专门的中文修改文档，同时同步更新相关资料文档。

更新文件：

- `doc/报警第一阶段优化变更总览.md`：在中文主文档列表和当前边界中追加 `agent.md` 与本轮修改文档的索引说明。

无变化。

本轮没有新增配置开关。

1. 删除 `agent.md`。
2. 删除 `doc/本轮修改记录-20260523-agent规范与文档追加.md`。

本轮为文档追加，主要验证点：

1. 确认 `hpis-alarm` 模块根目录原本没有 `agent.md` 或 `AGENTS.md`。

## 未覆盖风险

1. 本文件只能约束后续协作流程，不能自动保证所有 Agent 都会执行；后续提交前仍需要人工或 review 检查。
2. “详细中文注释”不能变成无意义注释，后续 review 应重点检查注释是否解释风险边界、失败路径和回滚语义。
3. 如果未来一次修改跨多个模块，当前 `agent.md` 只覆盖 `hpis-alarm`，其他模块需要另行补充模块级规范。
