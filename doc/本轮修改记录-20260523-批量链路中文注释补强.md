# 本轮修改记录：批量链路中文注释补强

## 修改背景

前两轮改动引入了 stop worker 空闲暂停、SQL 降噪、insert/stop 批量化、自研 MQ start 聚合、Spring AMQP consumer batch 等链路。代码已经具备功能，但部分新增类和函数的风险边界仍靠口头说明或文档理解，不利于后续维护。

本轮按 `agent.md` 要求，对前两轮涉及的关键类、函数、配置和 Mapper 增加中文注释，重点说明为什么这样做、失败边界、幂等边界、事务边界和回滚语义。

## 修改范围

本轮只增加或调整注释，不改变业务逻辑、方法签名、SQL、配置默认值或数据库结构。

主要补充注释的代码：

- `AlarmBatchProperties`：补充批量开关、非法值兜底、consumer batch、自研聚合、prefetch/concurrency 风险说明。
- `AlarmInsertAggregateResult`：补充 SUCCESS/SKIP/DROP/FAIL 对 MQ ack/nack 的含义。
- `AlarmInsertAggregateService`：补充自研聚合的 Channel 线程边界、QUEUED/FLUSHING 状态边界、停机 drain、单条兜底和 Redis 去重复用说明。
- `AlarmInsertBatchConsumeService`：补充 consumer batch 返回结果下标对齐、prepare 与批量持久化边界、批量失败拆单兜底说明。
- `RabbitMQAlarmBatchListener`：补充 consumer batch 容器配置、混合 operCode 分流、逐条 ack/nack、坏消息 reject 和 stop/status 旧语义说明。
- `RabbitMQAlarmListener`：补充旧单条 listener 作为默认/回滚路径、自研聚合结果确认和 stop 可靠落库说明。
- `AlarmStopWorkerSignal`、`AlarmStopEventWorker`：补充 idle pause、低频 probe、唤醒幂等、scheduled 短路边界说明。
- `AlarmCidIndexService`、`AlarmCidIndexMapper`：补充批量 route 查询、hot/stale、CLOSED 幂等和 ROUTE_MISSING 语义说明。
- `AlarmStopEventService`、`AlarmStopEventMapper`：补充 stop event 可靠入口、批量 route 查询、route 缺失失败、批量异常单条兜底和 side effect 不阻塞核心消警说明。
- `AlarmStopSideEffectService`、`AlarmStopSideEffectMapper`：补充副作用事件只落库、不直接阻塞核心消警、批量 upsert 和批量 DONE 边界说明。
- `AlarmServiceImpl`、`IAlarmService`：补充 `insertAlarms`、prepared context、批量事务、单条兜底、Redis 去重释放、push afterCommit 兼容说明。
- `AlarmMapper`、`AlarmElectrolyticCellMapper`、`AlarmPartialDischargeMapper`、相关 service 接口：补充主表/扩展表批量插入必须与主报警同事务的说明。

## 幂等风险

无新增运行期幂等风险。

本轮注释重点标明了以下幂等边界：

- `SUCCESS/SKIP/DROP/FAIL` 与 MQ ack/nack 的关系。
- Redis 断线去重只在 prepare 阶段执行一次。
- 批量失败拆单必须复用 `AlarmInsertContext`。
- stop route CLOSED 视为重复 stop 幂等成功。
- route 真缺失标记 `FAILED/ROUTE_MISSING`。

## 数据一致性风险

无新增数据一致性风险。

本轮注释明确了以下一致性边界：

- stop 先落 `alarm_stop_event` 后 ack。
- start 批量入库后立即批量补偿 stop 早到。
- insert 批量主表、处理表、扩展表、cid route 必须在同一事务内。
- stop 批量顺序是业务表 endTime、cid route CLOSED、stop event APPLIED。
- side effect 失败不反向影响核心消警闭环。
